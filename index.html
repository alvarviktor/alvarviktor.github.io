File Transfer Application

Previous Next

A windows desktop application made using C++ that demonstrates the differences between TCP and UDP under various conditions. The application accomplishes this by sending files of varying sizes between two computers in a LAN and collecting data such as packet loss and transfer speed.

The user must select the protocol they wish to use, TCP or UDP. Afterwards, they can choose to run the application as a client or server. The server must be running before the client can connect to it, then the client must enter in the server's IP address, port number, and the size and amount of packets they wish to send.

After experimenting with the application in a LAN, I found that transfering data using the TCP protocol is much more reliable and has fewer missing packets. Meanwhile, transfering data using UDP is much quicker, but results in many lost packets when sending lots of data.
